## 每段新筆記的品質門檻(所有草擬/查核 agent 的共同標準)
1. **`h`**:精簡小標,點出該段的「子觀念」(與同主題其他段不重複)。
2. **`points`**:2–4 條;每條 **能單獨被 TTS 唸順** —— 不放裸 LaTeX、不放只在視覺上才懂的符號堆;數學用「平方/根號/分之」等可唸字詞輔助,必要時把純符號放 `formula`。
3. **`trap`**:該段有常見錯誤就**一定寫**(會考易錯點、定義混淆、單位、符號)。
4. **`formula`**:理工科(數/自)有公式就放這裡(符號式),與 points 的文字說明分工。
5. **`svg`+`caption`**:**擇要**——只在「圖比字清楚」(數線/流程/結構/時間軸/幾何/座標)時才畫。畫就守規則:根元素只給 `viewBox` 不給 width/height;線條文字用**深色固定 hex**(墨 `#3a342e`、珊瑚 `#c9755c`、綠 `#5aa874`、警示 `#f0a52e`),**禁** `currentColor`/白色/`<image>`/`<script>`/外部 URL(離線+深色模式可見)。
6. **正確性**:會考(108 課綱、國中程度)範圍與難度;史實年代、公式、定義經獨立對抗式查核才留。

## 動什麼／不動什麼
- ✅ 加新段落(附加在既有段落**之後**);理工科該補的 formula/caption。
- ❌ 不刪、不重寫既有筆記;不改 `topic.id` / `name` / `quiz`;`audioSrc` 維持空。

## 流程(每科一輪)
1. workflow:逐主題並行**草擬**新段落 → **對抗式查核**(挑錯:數學算錯、史地年代、定義、與既有段重複、TTS 友善、SVG 規則)→ 通過才收。
2. 我把查核後的新段落**外科式插入** data 檔(只在各主題 notes 陣列尾端加段,不重排既有內容 → diff 乾淨)。
3. 插入後跑語法檢查 + 重算段數確認;**一科做完停下給使用者看**,定方向再進下一科。
